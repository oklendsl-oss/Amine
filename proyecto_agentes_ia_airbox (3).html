01 — Situación actual

El diagnóstico

El departamento de Customer Care de AIRBOX opera hoy con 3 personas cubriendo tareas altamente repetitivas y sistematizables. La combinación de volumen predecible y herramientas digitales ya instaladas (Outlook, SAP Business One, Excel) hace de este departamento el candidato ideal para una primera automatización.

📧

Comunicación con clientes

Respuesta a consultas, confirmaciones de pedido y seguimiento vía Outlook. Alto volumen, baja complejidad media.

📦

Gestión de pedidos

Alta y consulta de pedidos en SAP Business One. Proceso manual y repetitivo con pasos definidos y predecibles.

⚠️

Reclamaciones

Gestión de incidencias: botellas defectuosas, retrasos, errores de factura. Requiere consulta de historial en SAP.

🚚

Rutas de chóferes

Planificación de rutas de reparto de botellas a bomberos y empresas mediante hojas Excel. Proceso diario de alta frecuencia.

02 — Tecnología

El stack tecnológico

Cuatro herramientas con roles bien diferenciados. No compiten entre sí: cada una resuelve una parte distinta del sistema. NemoClaw actúa como plataforma base de seguridad empresarial; el resto son los motores de inteligencia y ejecución.

Plataforma base · Seguridad
NemoClaw

Plataforma de agentes IA empresarial de NVIDIA, construida sobre OpenClaw con guardrails de seguridad integrados. Bloquea acciones destructivas, requiere confirmación humana para operaciones críticas, registra auditoría completa. Datos siempre en tu infraestructura local. Lanzada en GTC 2026.

Orquestador
Motor de razonamiento principal
Claude (API)

Cerebro de razonamiento de alta calidad para redacción de respuestas a clientes, clasificación de reclamaciones, análisis de situaciones complejas y generación de instrucciones para los demás agentes. Se conecta vía API desde NemoClaw.

Razonamiento
Motor de contexto largo
Kimi K2.5

Especializado en ventanas de contexto extremadamente largas. Ideal para analizar el historial completo de un cliente, procesar documentos de rutas con muchas variables o revisar expedientes de reclamaciones extensos sin perder información.

Contexto largo
Ejecución en escritorio
Claude Cowork

Agente de escritorio que opera aplicaciones como lo haría un humano. Necesario cuando SAP Business One no tiene API accesible o cuando hay que interactuar con la interfaz gráfica directamente. Complementa a NemoClaw para acciones en sistemas sin API.

Ejecución UI

04 — Agentes IA

Los 4 agentes en detalle

Cada agente tiene un dominio claro, usa el modelo más adecuado para su tarea, y está configurado con guardrails específicos en NemoClaw para que no tome decisiones fuera de su ámbito.

01

Agente de Atención

Motor principal: Claude API via NemoClaw

  • Lee emails entrantes en Outlook y clasifica la intención
  • Responde consultas de precio, estado de pedido y dudas técnicas
  • Gestiona el historial de conversación por cliente
  • Escala al supervisor si la solicitud es ambigua o sensible
  • Envía confirmaciones y acuses de recibo automáticos
Outlook API Claude (razonamiento) NemoClaw guardrails
02

Agente de Pedidos

Motor: Claude + Claude Cowork para SAP

  • Extrae datos de pedido del email o chat del cliente
  • Verifica disponibilidad de stock en SAP Business One
  • Crea órdenes de venta vía SAP Service Layer (API REST)
  • Si la API no está disponible, opera la interfaz SAP con Cowork
  • Pedidos superiores a umbral de importe → aprobación supervisor
SAP Business One SAP Service Layer Claude Cowork (UI)
03

Agente de Reclamaciones

Motor: Claude + Kimi K2.5 para historial largo

  • Categoriza la incidencia: defecto, retraso, error de factura
  • Consulta historial completo del cliente en SAP (Kimi para contexto largo)
  • Genera borrador de respuesta con propuesta de resolución
  • Clasifica por gravedad y escala si supera umbral definido
  • Nunca toma decisiones económicas sin aprobación humana
SAP Business One Kimi K2.5 Claude API
04

Agente de Rutas

Motor: Claude API + lectura/escritura Excel

  • Lee pedidos de entrega pendientes desde SAP Business One
  • Consulta el Excel de rutas: zonas, chóferes, capacidad de carga
  • Asigna paradas a cada chófer optimizando por zona geográfica
  • Actualiza la hoja Excel con la ruta del día para cada conductor
  • Genera resumen y lo envía por email a cada chófer vía Outlook
Excel Rutas SAP Business One Outlook (envío)

05 — Implementación

Fases del proyecto

Cuatro fases que van de menos a más autonomía. El principio rector: nunca desplegar autonomía completa sin que la fase anterior esté validada. Cada fase tiene criterios de éxito antes de avanzar.

Fase 01 Semanas 1–3

Infraestructura y piloto de atención

Instalación de NemoClaw en el servidor de AIRBOX, conexión con Outlook y configuración básica de seguridad (permisos por módulo, log de auditoría, panel de supervisor). El agente de atención entra en modo asistido: genera respuestas para revisión y aprobación humana antes de enviar. Cero automatización completa, solo asistencia.

Instalar NemoClaw on-premise Conectar Outlook API Configurar Claude API Panel supervisor 10 plantillas de respuesta Test con emails reales
Fase 02 Semanas 4–8

Integración SAP y agente de pedidos

Verificación de la versión de SAP Business One y disponibilidad de Service Layer (API REST). Si está disponible, se construye la integración directa. Si no, Claude Cowork opera la interfaz gráfica. Se activa el agente de pedidos para los 5 tipos de pedido más frecuentes, con umbral de aprobación automática por importe. El agente de atención pasa a modo semi-autónomo (responde sin revisión previa en categorías de bajo riesgo).

Auditar SAP Service Layer Mapear tipos de pedido Umbral aprobación automática Test pedidos en sandbox SAP Atención semi-autónoma
Fase 03 Semanas 9–13

Reclamaciones, rutas y autonomía completa

Se despliegan los agentes de reclamaciones (con Kimi K2.5 para historial largo) y rutas (con lectura/escritura Excel). El agente de rutas genera la planificación diaria de forma autónoma cada mañana a las 7:00. El supervisor recibe el resumen y puede modificar antes de que se envíe a los chóferes. En esta fase los agentes operan de forma autónoma en sus dominios y el supervisor gestiona solo excepciones.

Agente reclamaciones Integrar Kimi K2.5 Agente rutas automático Cron job diario 7:00 Dashboard excepciones
Fase 04 Semanas 14–16

Estabilización y traspaso

Periodo de ajuste fino: revisión de los casos que escaparon a los agentes, calibración de umbrales de escalado, documentación del sistema para el supervisor. La persona que asume el rol de supervisor recibe formación específica para gestionar el sistema: interpretar el panel de auditoría, ajustar instrucciones de los agentes y reportar incidencias. Cierre formal del proyecto piloto.

Ajuste de umbrales Documentación supervisor Formación KPIs de seguimiento Cierre piloto

06 — Impacto organizativo

El rol del supervisor humano

La inteligencia humana no desaparece: se concentra. La persona que permanezca pasa de operador reactivo a gestor estratégico del sistema. Su trabajo cambia profundamente.

Antes — 3 personas, tareas repetitivas

  • Leer y responder decenas de emails diarios
  • Dar de alta pedidos manualmente en SAP
  • Consultar el historial de cada cliente para cada reclamación
  • Actualizar el Excel de rutas cada mañana
  • Enviar confirmaciones y acuses de recibo
  • Buscar información de stock en SAP
  • Coordinar por teléfono cambios de ruta

Después — 1 supervisor, rol estratégico

  • Revisar el dashboard diario de acciones ejecutadas
  • Aprobar excepciones marcadas por los agentes
  • Gestionar reclamaciones graves o fuera de umbral
  • Atender el canal telefónico (único canal manual)
  • Ajustar instrucciones de los agentes cuando aparezcan casos nuevos
  • Reportar incidencias del sistema a IT o proveedor
  • Proponer mejoras y nuevos casos de uso para la siguiente fase

07 — Economía del proyecto

Estimación de costes

Costes orientativos. Las variables más importantes son el coste de integración SAP (depende de la versión instalada) y el consumo de tokens de los modelos (depende del volumen de mensajes). Ambos se estiman una vez auditado el sistema actual.

Concepto Tipo Estimación
NemoClaw — plataforma base Open source 0 €
Servidor / VPS on-premise Infraestructura ~50–150 €/mes
Claude API (Anthropic) Consumo por tokens ~100–300 €/mes
Kimi K2.5 API Consumo por tokens ~40–80 €/mes
Claude Cowork Licencia de escritorio Por confirmar
Integración SAP Service Layer Desarrollo / consultoría ~1.500–4.000 € (único)
Configuración y puesta en marcha Implementación ~2.000–5.000 € (único)
Formación supervisor Formación ~500 € (único)
Coste operativo mensual (tras implantación) ~200–550 €/mes

Nota: Frente al coste de 2 salarios eliminados (estimado ~40.000–60.000 €/año en costes totales de empresa), el ROI positivo se alcanza en los primeros meses de operación estable. El coste de tokens puede optimizarse significativamente usando modelos locales Nemotron para tareas simples y reservando Claude/Kimi para razonamiento complejo.

08 — Arranque

Próximos pasos inmediatos

Para arrancar el proyecto esta semana, solo hacen falta estas cinco acciones. No requieren inversión previa.

01

Auditar SAP

Verificar versión de SAP Business One instalada y si tiene Service Layer activa. Define si la integración es directa o requiere Cowork.

02

Asesoría laboral

Consultar a la asesoría laboral el procedimiento correcto para la reducción de plantilla antes de comunicar nada al equipo.

03

Instalar NemoClaw

Desplegar NemoClaw en un servidor de test de AIRBOX (puede ser un PC spare). Conectar Outlook y Claude API en modo sandbox.

04

Elegir supervisor

Decidir qué persona del equipo actual asume el rol de supervisor del sistema. Esta persona lidera el piloto junto al técnico que implemente.

05

KPIs de partida

Medir hoy: número de emails/día, tipos más frecuentes, pedidos/semana, reclamaciones/mes. Son la línea base para medir el ROI.
